像人出示护照一样，每个 Agent 在调用工具前都必须出示自己的“数字护照”，并证明：

* 我现在运行在哪

痛点

目前 Agent 系统最大的问题之一：

用户看到：

“Agent 帮我修改了 Salesforce 记录”

实际上没人知道：

* 是哪个 Agent？
* 是哪个版本？
* 代表哪个用户？
* 是否经过授权？
* 是否被 Prompt Injection 劫持？
* 是否在合法环境运行？

未来企业不会接受：

“GPT帮我干的”

而必须变成：

“Agent #123 在 AWS us-west-2 的可信工作负载workload中，代表 Kevin Zhu，通过审批后修改了这条记录。”

核心创新

1. Agent Passport

类似 JWT

{
  "agent_id": "finance-agent",
  "agent_version": "1.3.2",
  "owner": "OpenAI",
  "delegated_user": "kevin",
  "runtime": "aws-us-west-2",
  "expires": "5min",
  "allowed_tools": [
      "gmail.read",
      "calendar.read"
  ]
}

⸻

2. Runtime Attestation (pre-approval)

不仅验证 Token

还验证：

这个 Agent 是否运行在
被批准的环境中？

例如：

✅ AWS EKS

✅ Azure Container Apps

❌ 用户本地 Docker

⸻

3. Delegation Chain

显示完整责任链：

Kevin
  ↓ delegated
Executive Assistant Agent
  ↓ delegated
Calendar Agent
  ↓ delegated
Google Calendar API

类似：

git blame

但用于 Agent。
