# Phase 1: Define Demo Scope

## Objective

Finalize the demo narrative, audience, and agent roster before implementation starts.

## Tasks

### Task 1.1: Finalize Demo Narrative

Define the end-to-end story:

- Holiday travel surge
- Airport check-in systems fail
- Kiosks, mobile check-in, and bag-drop counters are affected
- Root cause is a firewall policy blocking identity token validation
- Agents recover service in about 12 minutes

Deliverable:

- Short narrative paragraph for the demo opening

### Task 1.2: Define Demo Personas

Define the people viewing or interacting with the system:

- Incident commander
- ServiceNow operator
- Security or compliance reviewer
- Platform engineer

Deliverable:

- Persona list with each person's goal during the demo

### Task 1.3: Define Agent Roster

Use these demo agents:

- Monitoring Agent
- ServiceNow Agent
- Cloud Agent
- Network Agent
- Identity Agent
- Firewall Agent
- Airport Ops Agent

Deliverable:

- Agent roster with one-line role descriptions

## Acceptance Criteria

- Demo story can be explained in under 60 seconds
- Business impact is clear
- Agent Passport value is clear
- Agent roster maps cleanly to the system design
